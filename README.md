# Дела рядом

Это **PWA-приложение** для сценариев микрозаработка: его можно открыть как мобильное приложение, добавить на главный экран, запускать в standalone-режиме и использовать базово офлайн.

## Что улучшено

- Современная главная с рекомендацией следующего действия.
- Поиск и фильтры в ленте заданий.
- Таймер выполнения задания и чек-лист качества.
- Серверное сохранение аккаунтов, заданий, баланса, истории операций и заявок поддержки; локально остаются черновики, UI-настройки и offline-кэш.
- Подключаемые backend-flow для подтверждения телефона SMS/push-кодом и личности через Госуслуги/ЕСИА, городская фильтрация заданий, адресные подсказки и вывод баланса.
- Лента активности, подробные уведомления по чатам/заданиям и бейдж непрочитанного.
- Расширенные фильтры ленты: категория, минимальная цена и расстояние.
- Чат поддержки: сначала отвечает бот, затем заявку можно передать оператору. Заявки и сообщения хранятся на backend; юзернейм `Поддержка_ДелаРядом358935-345324` открывает роль поддержки.
- Локальная админ-панель для модератора: список всех зарегистрированных пользователей, модерация заданий, жалобы/арбитраж, журнал действий и мониторинг ошибок.
- У каждого задания есть ID вида `DR-000001`, по которому поддержка может найти заявку и открыть чат задания.
- Фото-подтверждения выполнения через backend/S3-compatible storage без base64 в задачах, история финансовых операций, демо-push и блок доверия профиля.
- Улучшенная карта с более читаемыми подписями улицы и номера дома на маркерах.
- Улучшенный профиль, поддержка и сценарий сброса демо-данных.
- SEO/PWA-метаданные, robots.txt, подсказки по установке, карточки документов и backend-памятка для следующего этапа.

## Как открыть

Самый простой способ:

1. Откройте `index.html` в браузере.
2. Интерфейс сразу выглядит как мобильное приложение: полный экран, нижняя навигация, роли, лента, задания, чат и профиль.

## Как установить как приложение

Для настоящей установки PWA браузеру обычно нужен `https://` или `localhost`.

После запуска через локальный сервер или хостинг:

- Android/Chrome: меню браузера → **Установить приложение** или **Добавить на главный экран**.
- iPhone/Safari: кнопка **Поделиться** → **На экран “Домой”**.
- Desktop Chrome/Edge: иконка установки в адресной строке.

## Что входит

- `index.html` — приложение.
- `styles.css` — мобильный fullscreen-интерфейс.
- `app.js` — логика заданий, ролей, безопасной сделки, чата, поддержки, предупреждений, черновиков, статусов, таймеров, синхронизации с backend и PWA install prompt.
- Демо-роль модератора включается входом/регистрацией с юзернеймом `Поддержка_ДелаРядом358935-345324`.
- `manifest.webmanifest` — настройки установки приложения.
- `service-worker.js` — offline-кэш приложения.
- `app-icon.svg` — иконка приложения.
- `robots.txt` — базовые правила индексации для статического хостинга.
- `_headers` — заголовки для Netlify/статического хостинга: PWA-файлы не кэшируются слишком долго, статические ресурсы кэшируются безопасно.
- `BACKEND_INTEGRATION.md` — список серверных интеграций, которые нельзя честно заменить клиентским демо.
- `backend/server.js` — Node.js backend-каркас для OAuth через ЕСИА/Госуслуги.
- `.env.example` — шаблон переменных окружения для client ID, client secret и redirect URL.
- `config.js` — публичный runtime-конфиг фронта. Если backend на отдельном домене, укажите там `API_BASE_URL`.
- `render.yaml`, `railway.json`, `Procfile`, `Dockerfile`, `DEPLOYMENT.md` — файлы для деплоя backend-сайта на хостинг.

## Как запустить с backend OAuth

1. Установите Node.js 18+.
2. Скопируйте `.env.example` в `.env`.
3. Заполните ключи ЕСИА/Госуслуг.
4. В кабинете ЕСИА добавьте redirect URL: `http://localhost:3000/api/auth/esia/callback`.
5. Запустите `npm start` и откройте `http://localhost:3000`.

## Важно

Регистрация по телефону теперь подтверждается backend endpoint-ами `/api/auth/phone/start` и `/api/auth/phone/verify`. В локальном development-режиме backend возвращает одноразовый dev-код для теста; в production без реального SMS-провайдера код не имитируется.

Это MVP с backend-контуром. В клиенте заложены backend endpoints для ЕСИА, SMS/push, заданий, чата, поддержки, платежей, выплат, файлов и push-подписок, но реальные SMS/push, Госуслуги/ЕСИА, эскроу-счета, выплаты на банковские карты, безопасный ввод CVC/токенизация карты, GPS-доказательства, камера, push-уведомления, антифрод и арбитраж требуют договоров с провайдерами и защищённой серверной проверки.

Фотоотчёты больше не сохраняются как base64 внутри задач/сообщений. Для production задайте `FILE_STORAGE_PROVIDER=s3` и `S3_*` переменные: backend выдаёт короткую presigned PUT-ссылку для загрузки, а скачивание идёт через `/api/files/{fileId}/download` с проверкой владельца, участника задания, поддержки или admin. Если S3 не настроен, загрузка фото fail-closed и не подменяется небезопасным локальным base64.

Backend-каркас уже добавлен, но без ваших настоящих параметров ЕСИА, redirect URL и настроек в кабинете ЕСИА он будет возвращать ошибку конфигурации, а не имитировать вход.

После выбора постоянного домена добавьте `canonical`, OpenGraph URL и абсолютную строку `Sitemap:` в `robots.txt`.
